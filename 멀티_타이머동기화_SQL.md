# 멀티 타이머 동기화 — 서버 시간 함수

라운드 타이머가 사람마다 다르게 보이던 문제는, 각 브라우저가 **자기 기기 시계**로 남은 시간을 계산했기 때문입니다.
이제 **서버 시간**을 기준으로 맞춥니다. 아래 함수 하나만 추가하면 모든 계정의 타이머가 동일하게 흐릅니다.

Supabase → **SQL Editor → New query** 에 붙여넣고 **Run**:

```sql
create or replace function public.server_now()
returns bigint
language sql stable
as $$ select (extract(epoch from clock_timestamp()) * 1000)::bigint $$;
```

- 이 함수는 현재 서버 시각(밀리초)을 돌려줍니다.
- 각 브라우저는 접속 시 한 번 이 값을 받아 **자기 시계와 서버의 차이**를 보정합니다.
- 그래서 기기 시계가 조금 어긋나 있어도 라운드 시간이 모두 같게 보입니다.

> 이 SQL을 아직 안 돌렸어도 게임은 그대로 동작합니다(예전처럼 내 기기 시계 기준).
> 돌리고 나면 타이머가 정확히 일치합니다. 꼭 실행해 주세요.

> 참고: 진행 자체는 여전히 **방장 화면이 켜져 있어야** 돌아갑니다(방장이 라운드를 넘김).
> 방장이 탭을 닫으면 타이머가 멈춥니다. (자동 대체 진행은 추후 보완 예정)
