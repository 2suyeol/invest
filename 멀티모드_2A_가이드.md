# 멀티 모드 2-A (로비) — Supabase 설정

이번엔 **방(rooms)**과 **참가자(room_players)** 테이블을 만들고, **실시간(Realtime)**을 켭니다.
이걸 해야 방 만들기 / 참가 코드 입장 / 대기실 실시간 참가자 목록이 작동합니다.

## SQL 실행

Supabase → **SQL Editor → New query** 에 아래를 붙여넣고 **Run**:

```sql
-- 방
create table public.rooms (
  id          uuid primary key default gen_random_uuid(),
  code        text unique not null,
  host_id     uuid not null references auth.users on delete cascade,
  status      text not null default 'waiting',   -- waiting | playing | ended
  total_rounds int not null default 10,
  created_at  timestamptz default now()
);
alter table public.rooms enable row level security;
create policy "rooms_read"   on public.rooms for select to authenticated using (true);
create policy "rooms_insert" on public.rooms for insert to authenticated with check (auth.uid() = host_id);
create policy "rooms_update" on public.rooms for update to authenticated using (auth.uid() = host_id);
create policy "rooms_delete" on public.rooms for delete to authenticated using (auth.uid() = host_id);

-- 방 참가자
create table public.room_players (
  id        uuid primary key default gen_random_uuid(),
  room_id   uuid not null references public.rooms on delete cascade,
  user_id   uuid not null references auth.users on delete cascade,
  nickname  text not null,
  joined_at timestamptz default now(),
  unique (room_id, user_id)
);
alter table public.room_players enable row level security;
create policy "rp_read"   on public.room_players for select to authenticated using (true);
create policy "rp_insert" on public.room_players for insert to authenticated with check (auth.uid() = user_id);
create policy "rp_delete" on public.room_players for delete to authenticated using (auth.uid() = user_id);

-- 실시간(Realtime) 활성화
alter publication supabase_realtime add table public.rooms;
alter publication supabase_realtime add table public.room_players;
```

이 SQL이 하는 일:
- `rooms`: 방 정보(참가 코드, 방장, 상태, 라운드 수). 방장만 만들고/수정 가능.
- `room_players`: 누가 그 방에 들어와 있는지. 본인만 자기 행을 추가/삭제.
- 마지막 두 줄: 두 테이블의 변경을 **실시간으로 전파**하도록 켜는 설정. (이게 있어야 대기실 참가자 목록이 자동으로 갱신됨)

> 이미 한 번 실행했다면 다시 실행 시 "already exists" 오류가 날 수 있어요. 그 경우 무시하거나,
> 마지막 두 `alter publication ...` 줄만 따로 실행하면 됩니다.

## 테스트 방법 (실시간 확인)

혼자서도 두 창으로 확인할 수 있어요:

1. 사이트를 **두 개의 창**에서 엽니다. (예: 일반 창 1개 + 시크릿/다른 브라우저 1개)
2. 각 창에서 **서로 다른 계정**으로 로그인.
3. 첫 번째 창: 멀티 모드 → **방 만들기** → 대기실에 **참가 코드**가 뜸.
4. 두 번째 창: 멀티 모드 → **방 참가** → 그 코드 입력 → 입장.
5. 그러면 첫 번째 창의 **참가자 목록에 두 번째 사람이 자동으로 나타나면** 실시간 성공! ✅
6. 방장(첫 번째 창)에서 라운드 수를 정하고 **게임 시작**을 누르면, 두 창 모두 "게임 시작" 화면으로 넘어갑니다.

> 지금은 "게임 시작"이 안내 화면까지입니다. 모두에게 동시에 굴러가는 라운드·주가·뉴스는
> 다음 단계(2-B)에서 연결합니다.

## 문제 해결

- **방 만들기 실패** → 위 SQL을 실행 안 했거나 권한(RLS) 누락. SQL 다시 실행.
- **참가자 목록이 자동으로 안 바뀜** → `alter publication supabase_realtime add table ...` 두 줄이
  실행됐는지 확인. (실시간 미활성 상태)
- **"violates row-level security"** → 정책 SQL이 빠짐. 해당 테이블 정책 부분을 다시 실행.
