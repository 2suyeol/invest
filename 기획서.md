# 페이퍼 거래소 — 게임 기획서

> 가상 종목으로 즐기는 모의 투자 게임. 뉴스 기사가 주가를 움직이고,
> 혼자(개인 모드) 또는 친구들과(멀티 모드) 겨룬다.

---

## 1. 전체 구조

로그인 후 **모드 선택** 화면이 뜬다.

```
[로그인/회원가입]
        │
        ▼
   [모드 선택]
   ┌──────────┬──────────┐
   │ 개인 모드 │ 멀티 모드 │
   └──────────┴──────────┘
```

- **개인 모드**: 혼자 모의 투자. 회원가입한 전체 유저와 점수로 랭킹을 겨룬다.
- **멀티 모드**: 친구들과 방을 만들어 정해진 라운드 동안 실시간으로 대결한다.

---

## 2. 개인 모드

- 기본 자금 **100만 원**으로 시작.
- **라운드 개념 없이 무한 진행**한다. 상단에 이전 결과 배너는 띄우지 않는다.
- **1분(60초)마다** 주가와 뉴스가 새로 갱신된다.
- 뉴스는 **코인 없이 자유롭게 열람**한다. 신문사·등급에 따라 정보의 질만 다르다
  (명확한 기사 / 애매한 기사 / 헛소문). 무엇을 믿을지는 플레이어가 판단한다.
- **진행 상황이 저장**된다(현금·보유종목·은행·시세). 다시 들어오면 이어서 진행.
- **상단 메뉴**로 화면을 전환한다. 현재 메뉴: **주식**. (메뉴는 앞으로 확장 예정)
  - 주식: 시세·매매·보유종목·은행·뉴스·거래내역
  - 랭킹은 "전체 유저"가 모여야 의미가 있으므로, 글로벌 랭킹이 가능한 **1단계(Supabase)**에서
    메뉴로 추가한다. (지금은 화면에서 제외)

> ⚠️ **코인 · 미니게임 · "게임 종료 후 순위 화면"은 멀티 모드 전용**이며 개인 모드에는 없다.

---

## 3. 멀티 모드

### 3-1. 입장 흐름

```
[멀티 모드 진입]
        │
        ▼
  [방 만들기 / 방 참가]
   ┌──────────┬──────────┐
   │ 방 만들기 │  방 참가  │
   └────┬─────┴────┬─────┘
        │          │
        ▼          ▼
   [대기실 생성]  [참가 코드 입력]
   참가코드 상단 표시      │
   참가자 목록 표시 ◄──────┘
        │
   (방장이 시작)
        ▼
   [게임 시작]
```

- **방 만들기**: 대기실로 이동. 상단에 **참가 코드** 표시. 참가자 목록에 들어오는
  사람들의 아이디가 실시간으로 나타난다. 방을 만든 사람(**방장**)만 게임을 시작할 수 있다.
- **방 참가**: 참가 코드 입력창 → 코드를 적으면 해당 대기실로 이동.

### 3-2. 게임 설정 (방장이 시작 전 설정)

- **라운드 수**: 최소 10라운드 ~ 최대 30라운드.
- 각 라운드 길이: **1분**.

### 3-3. 라운드 진행

- 시작 시 전원 **100만 원** 보유.
- 1분 동안: 주식 매수/매도 + 뉴스로 정보 수집.
- **라운드가 바뀔 때** 주가가 변동되고, 뉴스 기사도 새로 갱신된다.

### 3-4. 라운드 사이 미니게임 (코인 획득)

- 한 라운드가 끝나면 간단한 미니게임이 등장.
- 미니게임으로 **코인**을 번다.
- 코인 분배(※ 확인 필요 — 6절 참고):
  - 게임 시작 시 기본 **5코인** 지급
  - 미니게임 **1등 → 3코인 / 2등 → 2코인 / 그 외 → 1코인**

### 3-5. 코인과 뉴스 등급

코인은 **뉴스 기사를 구매**하는 데 쓴다. 뉴스는 정보의 질에 따라 3단계:

| 등급 | 정보의 질 | 코인 비용(예시) |
|------|-----------|----------------|
| 1단계 | 낮음 (애매한 힌트, 가끔 헛소문) | 무료~저가 |
| 2단계 | 보통 (방향성 암시) | 중간 |
| 3단계 | 좋음 (사건·수혜 업종 명확) | 고가 |

- 뉴스 기사는 **라운드마다 내용이 달라진다.**
- 현실에 여러 신문사가 있듯 **뉴스 회사도 여러 곳**이며, 회사마다 서로 다른 내용/논조의
  기사를 준다. (일부 회사는 신뢰도가 낮아 헛다리 기사를 줄 수 있다.)

## 4. 뉴스 → 주가 변동 메커니즘 (핵심)

게임의 심장. "기사 내용에 따라 특정 업종 주가가 오른다"를 이렇게 구현한다.

1. 모든 종목은 **업종(섹터)** 에 속한다. (제약, 항공, 에너지, 기술, 식음료, 친환경, 금융, 우주항공 …)
2. 매 라운드 시스템이 1~2개의 **사건(이벤트)** 을 고른다. 각 사건은 업종별 영향이 정해져 있다.
   - 예: `감염병 확산` → 제약 ↑↑, 항공 ↓↓, 식음료 ↑
   - 예: `유가 급등` → 에너지 ↑↑, 항공 ↓
   - 예: `AI 수요 폭발` → 기술 ↑↑
   - 예: `친환경 정책` → 친환경 ↑↑, 에너지 ↓
3. 이 사건들로부터 **뉴스 기사**가 생성된다. 단, 등급에 따라 드러나는 정도가 다르다.
   - 3단계: 사건과 수혜 업종·방향을 **명확히** 알려줌
   - 2단계: 방향만 **암시**
   - 1단계: **애매**하거나 가끔 가짜 사건(헛소문)을 섞음
4. 라운드가 끝나면 활성 사건의 업종 영향 + 약간의 무작위 변동으로 **주가가 갱신**된다.
   → 좋은 정보를 산 사람이 올바른 업종을 미리 매수해 수익을 낸다.

## 5. 공통 시스템 (개인·멀티 모두 적용)

### 5-1. 은행 (예금·이자)

- 주식 외에 **은행**이 있어 현금을 예치하면 **1분마다 이자**가 붙는다.
- 이자율은 고정이 아니라 **금리 뉴스에 따라 변동**한다.
  - `금리 인상` 사건 → 예금 이자율 ↑
  - `금리 인하` 사건 → 예금 이자율 ↓
- 즉 뉴스에 **은행 이자 관련 기사**가 등장하며, 그 내용이 실제 이자율에 반영된다.
- 안전하게 이자를 받을지(예금) vs 주식에 베팅할지의 **선택지**가 생긴다.

### 5-2. 뉴스 다양화

- 같은 기사가 반복되지 않도록, 각 사건마다 **여러 표현의 기사 문구**를 두고 무작위로 고른다.
- 신문사도 무작위로 배정되고, 직전과 **같은 사건이 연속으로 나오지 않도록** 회피한다.
- 결과적으로 매 갱신마다 뉴스 피드가 다르게 보인다.

## 6. 확인이 필요한 점 (열린 질문)

- **코인 "5개 지불"**: 작성 내용("처음 게임 시작 시 5개 지불")을 *기본 5코인 지급*으로 해석했음.
  맞는지? 아니면 무언가에 5코인을 내는 것인지?
- **미니게임 종류**: 어떤 미니게임? (반응속도, 숫자 맞히기, 클릭 경쟁 등) 1개부터 시작 권장.
- **개인 모드의 코인/뉴스 구매**: → **결정됨.** 개인 모드는 코인 없이 모든 뉴스를 무료로 열람.
  코인·미니게임은 멀티 모드 전용.
- **랭킹 기준**: 최종 자산 1개 기준인지, 수익률·여러 게임 누적도 볼지?
- **게임당 종목 수**: 현재 8종목. 더 늘릴지?

## 7. 개발 단계 & 기술 스택

| 단계 | 내용 | 필요한 것 |
|------|------|-----------|
| **0단계 (완료)** | 개인 모드 프로토타입 — 1분마다 시세·뉴스 변동, 무료 뉴스 열람, 끝이 없는 모의 투자 | GitHub Pages만으로 OK (정적, localStorage) |
| **1단계 (코드 완료)** | 진짜 로그인(이메일) + 어디서든 이어지는 저장 + 글로벌 랭킹 | Supabase (Auth + DB). 본인 프로젝트 생성·키 입력 필요 → `Supabase_연동_가이드.md` 참고 |
| **2단계 A (코드 완료)** | 멀티 로비: 방 생성 / 참가 코드 / 실시간 대기실 / 방장 시작 | Supabase Realtime. SQL은 `멀티모드_2A_가이드.md` |
| **2단계 B (예정)** | 모두에게 동시 진행되는 라운드·주가·뉴스 동기화 | Supabase Realtime |
| **2단계 C (예정)** | 라운드 사이 미니게임 + 코인 + 뉴스 등급 구매 | — |

**왜 단계를 나누나?**
- 멀티 모드는 "여러 사람이 같은 방에 실시간으로 모여 동기화"되어야 하므로, 정적
  사이트(GitHub Pages)만으로는 불가능하다. 서버 역할을 하는 **Supabase**가 필요하다.
- 다행히 프론트엔드(화면)는 계속 GitHub에 두고, 데이터·실시간 동기화만 Supabase가
  맡으면 된다. 0단계 코드의 거래·뉴스·주가 로직은 그대로 재사용된다.

**스택 정리**
- 프론트엔드: HTML/CSS/JavaScript → GitHub Pages 배포
- 백엔드(1·2단계): Supabase (Postgres + Auth + Realtime, 무료 티어)
- 가상 시세: 외부 주가 API 불필요 — 코드 내부에서 생성
