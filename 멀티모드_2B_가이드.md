# 멀티 모드 2-B (동기화 게임) — Supabase 설정

2-A(로비)에 이어, 이번엔 **방 전체가 같은 시세·뉴스로 동시에 플레이**하도록 만듭니다.
필요한 건 컬럼 추가 SQL **한 번**뿐입니다. (2-A SQL을 먼저 실행했어야 합니다.)

## SQL 실행

Supabase → **SQL Editor → New query** 에 붙여넣고 **Run**:

```sql
-- 방에 '공유 게임 상태'를 담을 칸 (라운드/시세/뉴스/금리)
alter table public.rooms add column if not exists game jsonb;

-- 참가자별 지갑 (현금/예금/보유종목)
alter table public.room_players
  add column if not exists cash     numeric not null default 1000000,
  add column if not exists bank     numeric not null default 0,
  add column if not exists holdings jsonb   not null default '{}'::jsonb;

-- 참가자가 '자기 지갑'을 수정할 수 있게 (2-A엔 없던 UPDATE 권한)
create policy "rp_update" on public.room_players
  for update to authenticated
  using (auth.uid() = user_id) with check (auth.uid() = user_id);
```

이 SQL이 하는 일:
- `rooms.game`: 방장이 계산한 **공유 게임 상태**(현재 라운드, 종료 시각, 시세, 뉴스, 금리)를 저장.
  모두 이걸 실시간으로 읽어 같은 화면을 봅니다.
- `room_players`의 `cash/bank/holdings`: **각자의 지갑.** 자기 행만 수정 가능.
- `rp_update` 정책: 매매·입출금으로 자기 지갑을 바꿀 수 있게 허용.

> `policy "rp_update" already exists` 오류가 나면 이미 만든 것이니 무시해도 됩니다.

---

## 작동 방식 (요약)

- **방장**의 화면이 1분마다 시세·뉴스를 계산해 `rooms.game`에 기록합니다(진행 담당).
- 나머지는 그 변화를 **실시간으로 받아** 똑같은 시세·뉴스·라운드/타이머를 봅니다.
- 각자 매매·예금하면 **자기 지갑**이 갱신되고, **랭킹 탭**에 모두의 총자산 순위가 실시간 표시됩니다.
- 정해진 라운드 수가 끝나면 **최종 순위 화면**이 모두에게 뜹니다.

> ⚠️ 진행은 **방장 화면이 켜져 있는 동안** 돌아갑니다. 방장이 탭을 닫으면 라운드가 멈춰요.
> (자동 대체 진행은 추후 보완 가능. 지금은 방장이 끝까지 켜두면 됩니다.)

---

## 테스트 방법

1. 사이트를 **두 창**(일반 + 시크릿)에서 **다른 계정**으로 로그인.
2. 한쪽에서 **방 만들기** → 코드 확인.
3. 다른 쪽에서 **방 참가** → 코드 입력 → 대기실 합류 확인.
4. 방장 창에서 라운드 수 정하고 **게임 시작**.
5. 양쪽 모두 같은 시세·뉴스·타이머가 보이는지 확인.
6. 각 창에서 서로 다른 종목을 매수 → **랭킹 탭**에서 자산 순위가 실시간으로 바뀌는지 확인.
7. 라운드가 다 끝나면 양쪽에 **최종 순위 화면**이 뜨면 성공 ✅

## 문제 해결

- **게임 시작 후 시세가 안 움직임** → 방장 창이 켜져 있어야 진행됩니다(방장이 진행 담당).
- **매매가 안 됨 / 권한 오류** → `rp_update` 정책 SQL이 빠짐. 위 SQL 다시 실행.
- **랭킹이 안 바뀜** → 2-A의 실시간 활성화(`alter publication ...`)가 되어 있어야 합니다.
- **컬럼 관련 오류** → 위 `alter table ... add column` SQL 미실행. 실행 후 새로고침.
